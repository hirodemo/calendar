# 外来カレンダーWebアプリ 設計図

## 1. 概要・目的
外来診療において、次回受診日の決定とそれに伴う処方日数の計算を直感的かつスムーズに行うための、シンプルなカレンダー型Webアプリケーション。

## 2. デザイン方針
- **シンプルで視認性が高い:** 診療の妨げにならないよう、情報量を絞り、日付や計算結果がパッと見てわかるデザイン。
- **レスポンシブWebデザイン:** PC（カルテ用端末）だけでなく、タブレットやスマートフォンなど、どのデバイスから見ても適切に表示されるレイアウト。

## 3. 技術スタック
- **使用言語:** HTML, CSS, JavaScript （フロントエンドのみで動作する軽量な構成）
- ※フレームワークの大掛かりな導入は避け、ブラウザだけですぐに動くようにする想定。

## 4. コア機能
### ① 特定の経過日数の明示（ハイライト表示）
カレンダー上で、本日を起点に以下の日数が経過した日付が「一目でわかる」ように表示（色分け、バッジ、アイコンなど）を行う。
- 本日
- 7日後
- 14日後
- 21日後
- 28日後
- 42日後

### ② 処方日数の自動計算
- 医師がカレンダー上の任意の日付（次の外来日）をクリック・選択する。
- 選択した外来日に対して、**「今日を含めて何日分の処方薬を出せばよいか」**を自動で計算し、大きく表示する。
  - *計算式： (選択した日付 - 本日の日付) + 1日（当日分）*

## 5. 今回は「見送る・不要」とする機能（シンプル化のため）
初めてサイト制作する人が着実に実装できるよう、複雑な処理はあえて省きます。

1. **ViteやReactなどのモダンなビルドツール・フレームワーク**
   - **理由:** 環境構築だけでハードルが高くなるため。今回は `index.html`, `style.css`, `script.js` の3ファイルだけで完結する昔ながらのシンプルな構成にします。
2. **休診日（祝日・土日）の自動判定や入力エラーチェック**
   - **理由:** 祝日の判定は「国民の祝日」の対応などプログラムが急激に複雑になります。まずは「選んだ日との差分を計算する」だけの基本機能に絞ります。休診日でも選べてしまう状態であえてOKとします。
3. **カレンダーの高度な祝日判定等**
   - **理由:** 祝日の判定は「国民の祝日」の対応などでプログラムが急激に複雑になります。今回は「土日の色分け」程度に留め、祝日でも選べる状態にします。

## 6. 着実に実装していくためのステップ（提案）

**Step 1: 画面の骨組みとカレンダーUIの実装（HTML/CSS/JS）**
- 「今日」を起点として、カレンダー（月ごとのグリッド）を**1ヶ月ごと**画面上に直接描画する。
- カレンダー内に「本日」「14日後」「28日後」「56日後」「84日後」「6か月後」がパッと見て分かるようにハイライト（色付け・バッジ表示）をつける。

**Step 2: 処方日数の計算機能追加**
- カレンダー上の日付をタップ（クリック）すると、選んだ日付から「今日」を引き算して、「〇日分」と画面に大きく出す。

## 7. セキュリティと安全な公開・運用に向けて（超重要事項）
初心者がそのままインターネット上（GitHub Pagesやレンタルサーバーなど）に公開しても絶対に安全であるように、以下のルールを厳守して作ります。

1. **APIキーなどをコードに直接書かない（ハードコード禁止）**
   - **理由:** もし将来的に「祝日判定API」など外部の便利ツールを使うことになっても、HTMLやJavaScriptのファイル内にパスワードやAPIキーを直接書き込むと、世界中の誰からでも見えてしまい悪用される危険があります。
   - **対策:** 今回はまず「すべてブラウザ内だけで完結する（外部通信を行わない）」設計にするため、APIキー自体を使いません。これによりキー流出のリスクをゼロにします。もし使う場合はバックエンド（サーバー側）を用意する設計に変更します。
2. **患者情報などの個人情報を入力・保存させない**
   - **理由:** カルテ番号や氏名などを入力する欄を作ると、個人情報漏洩のリスクが発生します。
   - **対策:** 今回のアプリは「カレンダーの日付」のみを扱い、患者情報は一切入力しない「単なる日付の計算機」として設計します。
3. **外部から変なコードを入れ込まれないようにする（XSS対策）**
   - **理由:** 日付を選ぶだけのアプリでも、念のために入力された内容を画面に出す際は「そのまま出さずに無害化する」のが鉄則です。
   - **対策:** JavaScriptの `textContent` などの安全な命令だけを使って画面の書き換えを行い、意図しないプログラムが動かないようにします（`innerHTML` は原則使いません）。

まずはこのStep 1〜3と、セキュリティの鉄則を守りながら順にクリアしていくことで、絶対に外せない要素を確実に、かつ安全な形にすることができます。
